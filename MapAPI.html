API

Details about API

What

  • API stands for - Application Programming Interface
  • A set of protocols that allow different applications to communicate and exchange data.

Why

  • API helps to integrate the existing functionality with a new application.
  • Avoid code to be written from scratch
  • Helps in sharing data

Use case

    Let's assume, you want to create a food delivering app, like zomato or swiggy, and you want to integrate a map (google map) that shows the registered users of that app, how far the resturant is from the users location once a order is placed. Therefore, being an owner of the food delivering business, would you want to spend much time on creating a map service from the scratch or would you just prefer to use the already available service (google map service api), and focus on your core busiess?

    In my opinion, it is quite obvious to use a service which is already in the market. And when it comes to a map service, what would be a better choise than using a "google map API service".

    Let's figure it out how a Google map API can be integrated in our app (our website).

How to Implement an API?

  1. Go to google console
  2. Select a google map api
  3. Generate a token.
  4. Use the sample code provide by google API servie in your application.
